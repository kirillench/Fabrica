# Фабрика гипотез

MVP интеллектуального инструмента генерации и приоритизации исследовательских
гипотез для НИИ и промышленных лабораторий. RAG + LLM: система индексирует
базу знаний (статьи, патенты, отчёты), извлекает релевантные фрагменты и
формирует проверяемые гипотезы с обоснованием, ссылками на источники,
оценками (новизна / реализуемость / эффект / риск) и дорожной картой проверки.

## Архитектура

```
frontend/  React + Vite + Tailwind — liquid glass UI, тёмно-серая тема
backend/   FastAPI + ChromaDB — парсинг документов, RAG, LLM-пайплайн, экспорт
backend/domains/  Доменные пакеты (YAML): онтология, словари, шаблоны
docker-compose.yml  Neo4j для графа знаний (опционально)
.env       Настройки LLM (OpenAI-совместимый API) и Neo4j
```

### Масштабируемость на новые предметные области

Ядро системы не содержит доменных знаний. Предметная область описывается
YAML-файлом в `backend/domains/`: типы сущностей, словари терминов,
промпт-подсказки, типовые шаги экспериментов. Подключение новой области
(полимеры, композиты…) = новый YAML-файл, без изменения кода.

### Граф знаний

При индексации из документов извлекаются сущности и связи (гибрид:
словари домена всегда + LLM-обогащение при наличии ключа). Граф хранится
в Neo4j (`docker compose up -d neo4j`), а если сервер недоступен —
автоматически во встроенном хранилище (networkx + JSON): система
продолжает работать без инфраструктуры. Граф используется при генерации
(контекст-триплеты в промпте) и визуализируется на вкладке «Граф знаний».

### Безопасность (уровень 1)

`APP_TOKENS=токен:Имя,токен2:Имя2` в `.env` включает аутентификацию:
все API-запросы требуют `Authorization: Bearer`, каждый пользователь видит
только свои сессии (разграничение доступа), а файлы базы знаний отдаются
по короткоживущим HMAC-подписанным ссылкам (5 минут). Пустой `APP_TOKENS` —
режим разработки без аутентификации. Всё развёртывание локальное; данные
не покидают машину. Роадмап: шифрование в покое, роли, аудит-лог, SSO.

### Строгая обоснованность

Обоснования гипотез строятся только на загруженной базе знаний: векторная
выдача дополнительно фильтруется интерпретируемым правилом (общие доменные
сущности / лексические основы), гипотезы без подтверждённых источников
отбрасываются, каждая ссылка [S#] в тексте кликабельна и открывает
исходный документ.

## Быстрый старт

### 1. Настройка LLM (`.env` в корне)

Впишите свой ключ и модель — подойдёт любой OpenAI-совместимый провайдер
(OpenAI, DeepSeek, OpenRouter, локальная Ollama/vLLM):

```dotenv
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
```

Пока ключ пуст, работает **демо-режим** — мок-гипотезы, чтобы проверить UI
и пайплайн без LLM.

### 2. Бэкенд

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

При первой загрузке документа ChromaDB скачает модель эмбеддингов
(~80 МБ, один раз).

### 3. Фронтенд

```bash
cd frontend
npm install
npm run dev
```

Открыть http://localhost:5173

## Что умеет MVP

- **База знаний**: загрузка PDF / DOCX / XLSX / TXT / CSV / MD, чанкинг,
  векторная индексация (ChromaDB, локально)
- **Генерация гипотез**: RAG-поиск релевантных фрагментов → LLM формирует
  конкретные проверяемые гипотезы со ссылками [S1…Sn] на источники
- **Прозрачное ранжирование**: LLM оценивает 4 критерия, итоговый балл —
  взвешенная формула; веса настраиваются слайдерами («экспертная настройка»)
- **Обоснование**: механизм влияния, риски, цитаты источников с релевантностью,
  граф связей сущностей (материалы / процессы / параметры / свойства)
- **Дорожная карта проверки**: шаги экспериментов + критерий успеха/провала
- **Фидбэк-цикл**: вердикты экспертов (перспективна / отклонить) сохраняются
  и подмешиваются в промпт следующих генераций
- **Экспорт**: DOCX-отчёт с ранжированием, JSON, CSV (для Jira/YouTrack и БД)

## API (для интеграций)

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/health` | статус, демо-режим, модель |
| POST | `/api/documents` | загрузка файлов в базу знаний |
| GET / DELETE | `/api/documents[/{id}]` | список / удаление |
| POST | `/api/generate` | генерация гипотез |
| POST | `/api/feedback` | вердикт эксперта |
| POST | `/api/export` | отчёт docx / json / csv |

## Направления развития

- Мультиязычные эмбеддинги (`paraphrase-multilingual-MiniLM`) для смешанных
  RU/EN/ZH корпусов
- Полноценный граф знаний (Neo4j) с логическим выводом поверх сущностей
- Экспорт задач напрямую в Jira/YouTrack через их API
- Дообучение ранжирования на накопленном фидбэке (learning-to-rank)
