# Фабрика гипотез

Интеллектуальный инструмент генерации и приоритизации исследовательских
гипотез для НИИ и промышленных лабораторий. Система индексирует базу знаний
(статьи, патенты, отчёты, данные экспериментов), строит по ней граф знаний
и по запросу исследователя формирует **проверяемые гипотезы** с обоснованием,
ссылками на источники, оценками по четырём критериям и редактируемой
дорожной картой проверки.

Разработан на материалах реальной задачи: переработка хвостов
обогатительных фабрик (флотация цветных металлов).

---

## Проблема

На старте исследовательских проектов гипотезы рождаются вручную, на опыте
отдельных экспертов: результаты мозговых штурмов субъективны, накопленные
отчёты лежат мёртвым грузом в архивах, сравнить идеи по новизне, рискам
и ценности нечем, а старт проектов затягивается.

## Решение

| Проблема | Как решает «Фабрика гипотез» |
|---|---|
| Субъективность, зависимость от экспертов | Генерация на основе всей базы знаний, а не памяти одного человека; эксперт валидирует, а не вспоминает |
| Неиспользуемые архивы | Загрузка PDF/DOCX/XLSX/TXT → векторный индекс + граф знаний; каждый отчёт начинает работать |
| Нет прозрачной оценки | 4 критерия (новизна, реализуемость, эффект, риск) + открытая формула итогового балла с настраиваемыми весами |
| Необоснованные идеи | Строгий режим: гипотеза без подтверждения источниками из базы отбрасывается; каждое утверждение цитирует [S#] |
| Медленный старт проектов | От формулировки задачи до ранжированного списка гипотез с дорожными картами — за минуту |

## Ключевые возможности

- **Чат-сессии** — исследование ведётся тредами, как в ChatGPT: запрос →
  гипотезы → уточнение → новые гипотезы. Полная история сохраняется,
  включая правки дорожных карт и вердикты экспертов.
- **База знаний** — drag-and-drop загрузка PDF / DOCX / XLSX / TXT / CSV / MD,
  чанкинг, локальная векторная БД (ChromaDB), раздача исходников по
  подписанным ссылкам.
- **Граф знаний** — сущности (материалы, процессы, параметры, свойства,
  реагенты, оборудование) и связи извлекаются при индексации; граф
  визуализируется и подмешивается в контекст генерации. Хранилище — Neo4j,
  при его недоступности автоматический фолбэк на встроенное (networkx).
- **Прозрачное ранжирование** — LLM оценивает 4 критерия, итог считается
  открытой формулой; веса настраиваются слайдерами, направления можно
  исключать («экспертная настройка»).
- **Привязка к источникам** — ссылки [S#] в обосновании кликабельны и
  открывают исходный документ; у каждой цитаты — процент релевантности.
- **Конструктор дорожных карт** — шаги проверки редактируются инлайн
  (название, дни, стоимость, ресурсы), Гант-таймлайн, итоги по срокам
  и бюджету; правки попадают в экспорт.
- **Обучение на фидбэке** — вердикты экспертов (перспективна/отклонена)
  подмешиваются в промпт следующих генераций.
- **Экспорт** — DOCX-отчёт с ранжированием, JSON, CSV; как всего набора,
  так и каждой гипотезы отдельно.
- **Мультидоменность** — предметная область описывается YAML-пакетом;
  подключение новой области не требует изменения кода.

### Строгая обоснованность

Обоснования строятся только на загруженной базе знаний:

1. Векторная выдача дополнительно фильтруется интерпретируемым правилом
   (общие доменные сущности / лексические основы) — офф-топик запросы
   получают честный отказ, а не выдуманные гипотезы.
2. Промпт требует ссылку [S#] на каждое фактическое утверждение.
3. Гипотезы без подтверждённых источников отбрасываются на бэкенде.
4. Словари домена понимают анонимизацию отчётов («элемент 28» = никель).

### Доменные пакеты

`backend/domains/*.yaml` — типы сущностей, словари терминов со
словоформами, промпт-подсказки, типовые шаги проверки со стоимостью.
В комплекте: обогащение руд (основной), металлургия сплавов, полимеры
и композиты. Новый домен = новый YAML-файл.

### Безопасность (уровень 1)

- `APP_TOKENS` в `.env` включает аутентификацию: Bearer-токен на всех
  эндпоинтах, у каждого пользователя — только свои сессии.
- Файлы базы знаний — по короткоживущим HMAC-подписанным ссылкам (5 мин).
- Полностью локальное развёртывание: данные не покидают машину
  (эмбеддинги считаются офлайн).
- Роадмап: шифрование в покое, роли, аудит-лог, SSO института.

### Надёжность LLM-контура

Клиент OpenAI-совместимый: провайдер и модель задаются в `.env` без правки
кода. При 429/5xx/невалидном ответе — автоматический фолбэк по цепочке
запасных моделей (`LLM_FALLBACK_MODELS`). Без ключа система работает
в демо-режиме с честной пометкой в интерфейсе.

## Быстрый старт

Одной командой (при первом запуске сам создаст окружения):

```bash
./start.sh
```

Или по шагам:

### 1. Настройка `.env`

```dotenv
LLM_API_KEY=...                          # ключ провайдера
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=nvidia/nemotron-3-super-120b-a12b:free
LLM_FALLBACK_MODELS=google/gemma-4-31b-it:free
APP_TOKENS=токен1:Исследователь          # пусто = без аутентификации
```

### 2. Бэкенд

```bash
cd backend
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --port 8000
```

### 3. Фронтенд

```bash
cd frontend
npm install
npm run dev          # http://localhost:5173
```

### 4. Neo4j (опционально)

```bash
docker compose up -d neo4j   # браузер графа: http://localhost:7474
```

Без Neo4j граф знаний работает на встроенном хранилище.

## Сценарий демонстрации

1. Войти по токену → загрузить отчёты во вкладке «База знаний»
2. Сформулировать задачу: «снизить потери меди с хвостами флотации»,
   ограничения: «без замены основного оборудования»
3. Получить ранжированные гипотезы → раскрыть обоснование → кликнуть [S1],
   убедиться, что открывается исходный отчёт
4. Отредактировать дорожную карту (сроки/бюджет) → скачать DOCX
5. Вкладка «Граф знаний»: связи гранулометрия—флотация—извлечение
6. Поставить вердикты → следующая генерация учитывает фидбэк
7. Уточнить запрос в той же сессии — история треда сохраняется

## API

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/health` | статус, режим, модель (открыт) |
| GET | `/api/me` | текущий пользователь |
| GET | `/api/domains` | доменные пакеты |
| POST / GET / DELETE | `/api/documents[/{id}]` | база знаний |
| GET | `/api/documents/{id}/link` | подписанная ссылка на файл |
| GET | `/api/graph` | срез графа знаний |
| POST | `/api/generate` | генерация гипотез (в сессию) |
| CRUD | `/api/sessions…` | сессии-треды |
| PATCH | `/api/sessions/{sid}/hypotheses/{hid}` | правки карты/вердикт |
| POST | `/api/feedback` | вердикт эксперта |
| POST | `/api/export` | DOCX / JSON / CSV |

## Ограничения и развитие

- Схемы-изображения (PNG) пока не индексируются — в планах
  vision-извлечение
- Эмбеддинги англоцентричные — компенсировано лексическим фильтром;
  план: мультиязычная модель эмбеддингов для RU/EN/ZH корпусов
- Экспорт задач напрямую в Jira/YouTrack через их API
- Learning-to-rank на накопленных вердиктах экспертов
- Шифрование данных в покое, роли и аудит
